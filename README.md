# 🎮 Bullet vs Dead

**Bullet vs Dead** — это top-down шутер (вид сверху), вдохновлённый классическими играми про выживание против волн зомби. Игрок управляет персонажем с видом сверху, целится мышью и отбивается от всё усиливающихся орд врагов.

---

## Концепция игры

* Камера: вид строго сверху (90°)
* Управление:

  * **WASD** — движение
  * **Мышь** — прицеливание
  * **ЛКМ** — стрельба
* Игрок всегда направлен в сторону курсора
* Основная цель — выживать как можно дольше, уничтожая волны зомби

---

## Основные механики

### Игрок

* Движение по карте (WASD)
* Поворот к курсору мыши
* Стрельба в направлении прицела
* Система здоровья (HP)

---

### Стрельба

* Пули создаются при клике мыши
* Летят в сторону курсора
* Уничтожают врагов при попадании

---

### Зомби

* Появляются волнами
* Двигаются напрямую к игроку
* Наносят урон при столкновении

---

### Система волн

* После уничтожения всех врагов начинается новая волна
* Каждая новая волна:

  * больше врагов
  * выше сложность

---

## Камера

В игре используется **следящая камера**:

* Игрок всегда находится в центре экрана
* Мир смещается относительно игрока
* Все объекты имеют мировые координаты

Принцип:

* Камера = смещение (offset)
* Все объекты рендерятся с учётом позиции камеры

## Архитектура проекта

Текущая структура репозитория:

```
Bullet-vs-Dead/
│
├── main.py            # запуск игры, игровой цикл, камера, HUD
├── settings.py        # настройки и константы (скорости, HP, волны, цвета)
│
├── player.py          # логика игрока (движение, прицел, HP, урон)
├── enemy.py           # зомби (движение к игроку, HP, отрисовка)
├── bullet.py          # пули (полёт, попадания, исчезновение)
│
├── wave_manager.py    # система волн (спавн, счётчик волны, переход к следующей)
├── collision.py       # столкновения пуль с врагами и врагов с игроком
│
├── utils.py           # (планируется) вспомогательные функции
│
├── assets/            # (планируется) изображения, звуки
└── README.md
```

### Назначение модулей

| Файл | Ответственность |
|------|-----------------|
| `main.py` | Инициализация Pygame, цикл кадра, класс `Camera`, связка всех систем |
| `settings.py` | Единое место для баланса и визуальных констант |
| `player.py` | WASD, поворот к мыши, здоровье, кулдаун после удара |
| `enemy.py` | Один зомби: преследование игрока, урон, смерть |
| `bullet.py` | Снаряд от ЛКМ, проверка дистанции полёта |
| `wave_manager.py` | Старт волны, спавн врагов за экраном, рост числа зомби |
| `collision.py` | `check_bullet_enemy_collisions`, `check_enemy_player_collisions`, очистка пуль |

`main.py` не содержит игровую логику врагов и столкновений — только вызывает методы из модулей выше.

---

## Игровой цикл

Каждый кадр в `main.py` выполняется:

1. Обработка ввода (клавиатура + мышь, стрельба)
2. Обновление игрока, камеры, врагов и пуль
3. Проверка столкновений (`collision.py`)
4. Переход на следующую волну, если все враги уничтожены (`wave_manager.py`)
5. Отрисовка и HUD (HP, номер волны)

## План разработки

### Этап 1 (База)

* Движение игрока
* Поворот к мыши
* Стрельба

### Этап 2

* Добавление зомби
* Урон и столкновения

### Этап 3

* Баланс сложности

* Эффекты
* Улучшение ощущений от стрельбы
